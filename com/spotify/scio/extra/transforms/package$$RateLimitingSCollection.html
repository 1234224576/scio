  21. def withRateLimit(maxElementsPerSecond: Double): SCollection[T]

    Rate limit the number of elements this step will process per second.

    Rate limit the number of elements this step will process per second. Useful to rate limit throughput for a job writing to a database or making calls to external services. This is a way to attempt to make some surrounding steps obey the same rate limit however there are no guarantees.

    Note that this only limits the throughput for the current step as well as any steps running within the same machine (within a serialization boundary). Therefore any maps and filters surrounding this will be somewhat rate limited usually until a groupBy, reduce, or join (or anything that uses those under the hood e.g. distinct, fold) is hit.

    DUE TO THE ARCHITECTURE OF DOFNs IN BEAM, THIS DOES NOT GUARANTEE THAT A SURROUNDING MAP/TRANSFORM WILL ACTUALLY BE RATE LIMITED. IT FOR THE MOST PART WORKS HOWEVER YOU SHOULD BE ESPECIALLY CAREFUL. FOR BEST RESULTS, TURN OFF AUTOSCALING.

    maxElementsPerSecond

    The maxmimum number of elements which should be processed per second
